# CYCLOPS2.0-based Circadian Phase Estimation

This repository contains the final workflow and scripts used for CYCLOPS 2.0-based circadian phase estimation from expression matrices.

The codebase includes sample-level phase inference, donor-level aggregation through donor internal phase (DIP), and downstream visualization of expression-phase relationships. The repository is organized as a compact analysis workflow intended for reproducible execution rather than as a general-purpose software package.

## Repository organization

- `src/julia/cyclops_core.jl`: core CYCLOPS implementation used by this workflow
- `src/julia/run_phase_inference.jl`: Julia entry point for sample-level circadian phase inference
- `src/python/estimate_donor_internal_phase.py`: donor-level aggregation of inferred sample phases
- `src/python/plot_phase_fits.py`: visualization of core circadian gene expression against inferred phase
- `scripts/run_phase_pipeline.sh`: shell wrapper for processing a single dataset or multiple subdatasets
- `requirements.txt`: Python dependencies for downstream analysis and plotting
- `Project.toml`: Julia dependencies for phase inference

## Expected input files

Each dataset directory is expected to contain the following files:

- `expression.csv`: gene-by-sample expression matrix
- `seed_genes.txt`: optional reference seed list retained with the original analysis inputs
- `metadata.csv`: optional sample metadata used for cell-type stratified visualization

The first column of `expression.csv` should contain gene identifiers. Sample IDs in `metadata.csv`, when provided, should match the sample columns in `expression.csv` and the IDs written to the inferred phase output. The current public workflow reads the ordered gene list from the first column of `expression.csv`; `seed_genes.txt` is retained as an optional reference input.

A minimal input template is provided under `examples/input_template/` for format reference.

## Analysis workflow

### 1. Run sample-level phase inference

```bash
bash scripts/run_phase_pipeline.sh examples/input_template
```

By default, output is written under `results/`. If the input directory contains multiple subdirectories with `expression.csv`, the workflow processes each subdataset separately.

### 2. Estimate donor internal phase

```bash
python src/python/estimate_donor_internal_phase.py results/my_dataset_20260317_120000
```

This step aggregates inferred sample-level phases into donor-level circadian phase summaries and generates both tabular and graphical summaries.

If needed for downstream human-versus-mouse phase interpretation, the resulting donor-level phases can be shifted by 12 hours after inference.

### 3. Plot gene expression against inferred phase

```bash
python src/python/plot_phase_fits.py --results_path results/example_run --expression examples/input_template/expression.csv
```

If `--expression` is omitted, the script attempts to infer the corresponding `expression.csv` path from the repository layout, including the default timestamped `results/` directory names generated by the workflow. Passing `--expression` explicitly is still the most robust option when running outside the standard repository layout.

## Main outputs

The workflow produces the following representative outputs:

- `Fit_Output*.csv`: sample-level circadian phase inference output with sample IDs and inferred phases
- `phase_distribution.png`: global distribution of inferred phase values
- `clock_genes*.png`: phase-expression plots for selected clock genes
- `DIP_results.csv`: donor internal phase summary table
- `DIP_summary.png`: donor-level scatter and histogram summary
- `DIP_circular.png`: circular representation of donor internal phase
- `expression_vs_phase*.png`: fitted expression-versus-phase plots for selected genes

Additional downstream analyses can be performed on the inferred phase outputs as needed.
