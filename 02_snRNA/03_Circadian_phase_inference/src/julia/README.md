# Julia Phase Inference

This directory contains the Julia implementation used for sample-level circadian phase inference.

## Files

- `cyclops_core.jl`: bundled CYCLOPS core implementation used by this workflow
- `run_phase_inference.jl`: entry point for fitting circadian phase on one dataset directory

## Expected dataset structure

The Julia entry point expects a dataset directory that contains:

- `expression.csv`: gene-by-sample expression matrix
- `seed_genes.txt`: optional reference seed list retained with the original analysis inputs
- `metadata.csv`: optional sample metadata

`run_phase_inference.jl` reads gene identifiers from the first column of `expression.csv` and uses that ordered gene list during model fitting. The `seed_genes.txt` file is retained as an optional reference input, but it is not required by the current Julia entry point.

See `examples/input_template/` in the repository root for a minimal format example.

## Run

From the repository root:

```bash
julia --project=. src/julia/run_phase_inference.jl examples/input_template results/example_run
```

To enable more detailed progress messages:

```bash
CPI_VERBOSE=1 julia --project=. src/julia/run_phase_inference.jl examples/input_template results/example_run
```

## Main outputs

Typical outputs written by the Julia step include:

- `Fit_Output*.csv`: inferred sample-level phases
- `phase_distribution.png`: overall phase distribution plot
- additional diagnostic figures generated by the bundled CYCLOPS workflow

## Notes

- This directory is intentionally kept close to the final analysis code used in this workflow.
- The bundled core implementation was not heavily refactored during repository cleanup in order to minimize behavioral drift.
- Plot generation in the Julia entry point is best treated as workflow-level output rather than a reusable plotting API.
